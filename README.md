# Customer Issue & Account Assistant

Welcome to the **Customer Issue & Account Assistant** project. This repository contains a multi‑phase implementation of an AI‑assisted support system built with **Keycloak**, **FastAPI**, **React**, and **Postgres** (added in later phases).

---

## 📦 Prerequisites

- **Docker Desktop** (or Docker Engine) with Compose support
- **Node.js 20** (for local development, but Docker image provides it)
- **Python 3.12** (used in the backend Docker image)
- **git** to clone the repository
- Sufficient memory (≥ 2 GB) and free ports **8080**, **8000**, **3000**

## 🚀 Quick Start (Phase 1)

```bash
# 1️⃣ Clone the repo
git clone <repo‑url>
cd stage2-project

# 2️⃣ Copy example env file and adjust if needed
cp .env.example .env
# (optional) edit .env to change passwords or realm file location

# 3️⃣ Build and run the full stack with a single command
docker compose up --build
```

The stack will be reachable at:

| Service | URL | Notes |
|---------|-----|-------|
| **Keycloak admin console** | `http://localhost:8080` | Username/password from `.env.example` (`admin`/`admin`) |
| **FastAPI backend** | `http://localhost:8000/health` (health) and `http://localhost:8000/me` (JWT echo) |
| **React frontend** | `http://localhost:3000` | Redirects to Keycloak for login, then shows the `/me` response |

## 🐞 Troubleshooting

- **Container health checks failing** – Run `docker compose logs <service>` to see errors. Ensure Docker has enough memory (≥ 2 GB) and that ports 8080/8000/3000 are not already in use.
- **Keycloak realm not loaded** – Verify `keycloak/realm-export.json` exists and matches the volume mount in `docker-compose.yml`.
- **Frontend cannot reach backend** – The Vite proxy forwards `/me` and `/health` to `http://backend:8000`. Make sure the `backend` service is healthy (`docker compose ps`).
- **Auth token errors** – Confirm the `KEYCLOAK_JWKS_URL` env var in the backend matches the Keycloak container URL (usually `http://keycloak:8080/realms/assistant/protocol/openid-connect/certs`).
- **Docker compose config errors** – Run `docker compose config` to validate the YAML before starting containers.
