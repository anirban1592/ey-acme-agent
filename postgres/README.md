# PostgreSQL Service (Ephemeral)

This directory contains everything needed to run the **PostgreSQL** instance used by the application data store (Phase 2).

## What is seeded?
- The `init.sql` script (see `../postgres/init.sql`) creates the seven tables required by the app (`customers`, `statuses`, `roles`, `users`, `user_roles`, `issues`, `issue_updates`).
- It also inserts a minimal set of seed rows so that the backend can immediately query realistic data.
- Gap‑fixes from the progress tracker are applied (e.g., `users.keycloak_id` column, `issues.domain` check, `updated_at` trigger, etc.).

## Ephemeral behaviour
- The container mounts a **tmpfs** at `/var/lib/postgresql/data`. This means the data directory lives only in memory and is cleared whenever the container stops or is restarted.
- Because the data directory is empty on each start, PostgreSQL re‑executes `init.sql` on every container recreation, guaranteeing a fresh seed state.

## How to connect
```bash
# From your host (uses the port mapping defined in docker‑compose.yml)
psql -h localhost -U ${POSTGRES_USER:-postgres} -d ${POSTGRES_DB:-issuesdb}
```
- The default credentials are defined in `.env.example`:
  - `POSTGRES_USER=postgres`
  - `POSTGRES_PASSWORD=postgres`
  - `POSTGRES_DB=issuesdb`

## Testing the ephemeral nature
1. Start the service: `docker compose up -d postgres`
2. Insert a throw‑away row:
   ```sql
   INSERT INTO customers (name) VALUES ('TempCo');
   ```
3. Restart the container: `docker compose restart postgres`
4. Run the same query again – the temporary row will be gone, confirming the seed has been re‑applied.

## Troubleshooting
- **Container fails to start** – ensure Docker has enough memory (≥ 2 GB) and that port `5432` is free.
- **`init.sql` errors** – run the script manually against a fresh `postgres:16-alpine` container to see the exact error output.
- **Healthcheck failing** – `docker compose logs postgres` will show if `pg_isready` cannot connect; double‑check that the environment variables are correctly interpolated.
